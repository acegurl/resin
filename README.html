Vietnam Plastic Resin Market — setup & weekly update guide

This dashboard now reads its numbers from a separate data.json file. You host the page once; each week you only replace data.json. Everyone who has the link sees the new data on refresh — you never send a file again.

What's in this folder

index.html — the dashboard. Host it once; it almost never changes.
data.json — this week's data. This is the only file you replace each week.
converter.html — your weekly tool. Open it in any browser, drag your Excel in, and it makes a fresh data.json. Nothing is installed or uploaded; it all runs locally in your browser.
README.html — this guide.
Important: the dashboard must be opened over http:// or https:// (i.e. from a host), not by double-clicking index.html from your desktop. Browsers block data.json loading from file:// paths for security. That's exactly why we host it.

Part 1 — Host it once (GitHub Pages)

  1. Create a free account at github.com if you don't have one.
  2. Click + (top-right) → New repository. Name it e.g. resin-dashboard. Tick Add a README file, then Create repository.
  3. In the repo: Add file → Upload files. Drag in index.html and data.json (and optionally converter.html). Commit.
  4. Go to Settings → Pages → under Source pick Deploy from a branch, branch main, folder / (root), Save.
  5. Wait ~1 minute. You'll get a link like https://YOURNAME.github.io/resin-dashboard/. That's your permanent dashboard URL.
Privacy note: a free GitHub Pages site is public — anyone with the link can view it. If the pricing data shouldn't be public, use a private host instead (e.g. Cloudflare Pages or Netlify with password protection), or GitHub's paid private Pages. The upload steps are nearly identical.

Part 2 — Show it inside SharePoint (optional)

SharePoint can't run this interactive page on its own, but it can frame the hosted page so your team opens it where they already work.

  1. In SharePoint, open the team site → New → Page → pick a blank layout.
  2. Click the + to add a web part → choose Embed.
  3. Paste this, replacing the URL with your GitHub Pages link:
    <iframe src="https://YOURNAME.github.io/resin-dashboard/" width="100%" height="2200" style="border:0"></iframe>
  4. Publish the page. Your team opens the SharePoint page and sees the live dashboard inside it.
If the Embed web part is blocked or shows nothing: your SharePoint admin needs to add your host (e.g. github.io) to the allowed embed domains (HTML Field Security / script settings). Send them this guide — that's a one-time tenant setting.

Part 3 — Your weekly update routine

  1. Update Plastic_Resin.xlsx on your computer as usual.
  2. Open converter.html in your browser (double-click is fine for this tool).
  3. If any narrative changed (overview bullets, the PE/PP/PVC reports, regional figures, "largest drop" note), edit the text in the expandable section. Otherwise skip.
  4. Drag your updated Plastic_Resin.xlsx onto the drop box.
  5. Click Download data.json.
  6. Upload that new data.json to your repo, replacing the old one (GitHub: open the repo → Add file → Upload files → drop it → Commit).

Within a minute, everyone's dashboard shows the new week on refresh. The dashboard's dates, the "View in PDF" export, and all percentages update automatically from the new file.

How the numbers are computed

  • Prices & changes come straight from the raw sheet of your Excel (header on row 2, with Group and Product columns and dated columns).
  • Pre-war / YTD / YOY baselines are picked automatically: pre-war = 26 Feb 2026, YTD = prior 31 Dec, YOY = same date one year earlier. These shift correctly as time passes.
  • The chart window is the most recent ~54 dated columns.
  • Narrative text (overview, reports, regional, "largest drop" note) is editorial — you control it in the converter; the numbers never depend on it.
Keep converter.html bookmarked. That, plus your Excel, is all you need each week.
